# Azure AI Foundry Agentic Agent

> Generated by the [Azure Demo Builder](https://aka.ms/trainer-demo-deploy) | 2026-04-22

Deploy an Azure AI Foundry project with a multi-tool agentic AI agent powered by GPT-4.1-mini. Demonstrates Code Interpreter, File Search, and Custom Function Calling with keyless authentication via Managed Identity.

## Architecture

![Architecture Diagram](03-architect-diagram.png)

| Service | SKU | Purpose |
|---------|-----|---------|
| AI Foundry Account (Hub) | S0 / AIServices | Central AI services endpoint |
| AI Foundry Project | — | Agent workspace with data store connections |
| GPT-4.1-mini | Standard (30K TPM) | Backing model for the agent |
| Cosmos DB | Serverless | Agent thread/conversation storage |
| Storage Account | Standard_LRS | Agent file storage |
| AI Search | Basic | Agent vector store |
| Log Analytics + App Insights | PerGB2018 | Observability |

## Prerequisites

- Azure CLI v2.65+
- Azure Developer CLI (`azd`) v1.12+
- Bicep CLI v0.32+
- Python 3.10+
- Azure subscription with **Contributor** and **User Access Administrator** roles

## Quick Start

### 1. Clone and navigate

```bash
cd generated-scenarios/foundry-agentic-agent
```

### 2. Initialize and deploy

```bash
azd auth login
azd init
azd up
```

When prompted:
- **Environment name**: e.g., `demo`
- **Azure location**: e.g., `eastus2` (for supporting resources)
- The AI Foundry Account deploys to `swedencentral` by default for GPT-4.1-mini availability

### 3. Note the deployment output

After deployment, copy the `projectEndpoint` value from the output.

### 4. Set up the Python agent

```bash
cd src/agent
pip install -r requirements.txt
cp .env.sample .env
```

Edit `.env` and paste the `projectEndpoint` value.

### 5. Run the agent

```bash
python main.py
```

The agent runs a 3-turn demo conversation:
1. **Weather lookup** — calls `get_weather()` for Stockholm and Tokyo
2. **Inventory check** — calls `lookup_inventory()` for two SKUs
3. **Code execution** — uses Code Interpreter to calculate compound interest

## Security

- **Zero secrets**: No API keys or connection strings. All authentication uses Managed Identity + RBAC.
- **Storage hardened**: HTTPS-only, TLS 1.2, shared key access disabled, no public blob access.
- **RBAC principle of least privilege**: Project MI gets only the specific roles it needs on each data store.
